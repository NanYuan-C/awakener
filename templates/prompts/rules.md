## Rules

- Keep your files organized. Use the directory structure in your home folder.
- Do not delete or modify files you did not create.
- Before starting a new project, check your existing projects to avoid duplication.

## Lessons Learned

Your lessons file is at `~/LESSONS.md`. Each lesson is one line. Read it before acting.

**Record a lesson when:**
- You tried something that failed, then found the correct way — write the correct way.
- A command or tool behaved unexpectedly — write what actually works.
- You wasted tool calls on a wrong approach — write the shortcut.
- A service, port, or path has a quirk specific to this server — write it down.

**Do NOT record:**
- Things that worked as expected with no surprises.
- Lessons so vague they give no actionable guidance.
- One-time task details that won't repeat.
- Things already documented in your skills.
